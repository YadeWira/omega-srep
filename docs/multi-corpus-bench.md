# Omega SREP - multi-corpus comparative benchmark (F6.10)

Generated by `tests/multi_corpus_bench.sh` on 2026-04-27T17:14:20Z.

## Setup

- Host: `Linux 6.12.74+deb13+1-amd64 x86_64`, 4 cores
- Method (osrep / srep): `-m4` (rereads-input mode, lowest RAM)
- FA invocation: `-mrep` for the LZ-only filter, `-dup -mstoring` for dedup-only.
  FA archives include a directory index + per-file metadata, so its `out_size`
  carries an extra ~hundred bytes of overhead relative to the raw-match-stream
  produced by srep / osrep.
- RSS via `getrusage(RUSAGE_CHILDREN)`; wall-clock via `time.monotonic`.
- Round-trip asserted via `cmp` for streamish tools, and against the
  largest size-matching extracted file for fa archives.

## Headline

- Fork parity: `osrep-m4` and `srep392-m4` produce byte-equal
  archives on every corpus (ratio columns identical), confirming
  Omega tracks upstream SREP 3.92 byte-for-byte on the LZ-only
  path. The hash seed is the only divergence and it does not
  affect ratio.
- `-dup` win on long-range-dup corpus (synth-128M):
  decompress RSS drops 66% (52 MiB -> 18 MiB) for +0.28% archive bloat.
- `-dup` overhead on real-world tarball (data.tar 1.79 GiB):
  +0.19% archive bloat with no ratio win, since the
  workload has few long-range duplicates. -dup remains correct
  (round-trip OK) but offers no benefit here.
- Cross-impl validation: `fa011-dup` and `osrep-dup-m4` produce
  ratios within 0.5%-of-input on the long-range-dup corpus,
  confirming Omega's CDC + dedup matches the FA 0.11 reference
  family it is modeled on.

## Notes

- `osrep-m4` vs `srep392-m4`: same algorithm, same params. Should be
  equivalent on ratio (modulo the per-archive hash seed) and within ~10%
  on time and RAM. Confirms Omega is a non-divergent fork.
- `osrep-dup-m4` vs `osrep-m4`: shows what the F5 dedup pre-pass adds.
  Win condition: equal-or-better ratio on long-range-dup corpora
  (synth, doubled tarballs), at lower or equal decompress RAM.
- `fa011-rep` vs `osrep-m4`: cross-implementation sanity check on the
  shared LZ77-preprocessor lineage. Different pipeline (FA stores into
  a solid block; osrep emits a flat .osr); ratio is comparable on text,
  diverges on tarballs because FA's solid-block layout interacts with
  -dup differently than a single-file invocation.
- `fa011-dup` vs `osrep-dup-m4`: same algorithm family (CDC dedup),
  different impl. Validates that Omega's `-dup` mode produces a dedup
  ratio in the same neighborhood as the reference implementation it is
  modeled on.
