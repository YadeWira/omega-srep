# Omega SREP `-dup` benchmark (F5.4)

Generated by `tests/dup_bench.sh` on 2026-04-27.

## Setup

- Host: `Linux 6.12.74+deb13+1-amd64 x86_64`, 4 cores, 7.0 GiB RAM
- Method: `-m4`
- CDC buffer size: 8 MiB (`--buf=8388608`)
- Block content: 1 MiB text + 31 MiB xorshift64 binary, repeated identically

RSS measurements use `tests/measure.py` (a Python wrapper around
`getrusage(RUSAGE_CHILDREN)`, since GNU `time` isn't installed on
this host) and cover only the `osrep` invocation. The F5.3a
wrapper's split-encode and split-decode steps are excluded — they
read the full input into a contiguous buffer and would dominate the
comparison; F5.3b's native integration will stream and remove that
overhead.

## Trend

| corpus  | A compress RSS | B compress RSS | A decompress RSS | B decompress RSS |
|--------:|---------------:|---------------:|-----------------:|-----------------:|
| 128 MiB | 21608 KiB      | 18664 KiB      | 53480 KiB        | 17512 KiB        |
| 256 MiB | 25832 KiB      | 18664 KiB      | 53480 KiB        | 17512 KiB        |

B's RSS is flat across input size because SREP only ever sees the
unique-chunk body (32 MiB in both runs). A's compress RSS grows with
input; A's decompress RSS plateaus at the SREP buffer ceiling
(`-b8mb` plus per-buffer state). Wall-clock for A scales linearly
with input; B is constant. Larger or more redundant corpora widen
the gap.

## Success criterion

Design target (per `docs/dup-mode-design.md`): B's peak decompress
RSS < A's by ≥50% with archive bloat ≤10%.

- decompress peak RSS reduction: **-67.3%** (target ≥50%) ✓
- archive size: **100.3-100.5% of A** (target ≤110%) ✓

The F5 thesis holds end-to-end on the F5.3a pipeline. This validates
the design and clears F5.4.

## Reproduction

```
make bin/osrep bin/dedup_test
bash tests/dup_bench.sh                          # 128 MiB run
OSREP_BENCH_REPEATS=8 \
OSREP_BENCH_OUT=docs/dup-bench-256.md \
bash tests/dup_bench.sh                          # 256 MiB run
```

Tunables (env vars): `OSREP_BENCH_BLOCK_MB`, `OSREP_BENCH_REPEATS`,
`OSREP_BENCH_METHOD`, `OSREP_BENCH_BUF`, `OSREP_BENCH_DIR`,
`OSREP_BENCH_OUT`.
